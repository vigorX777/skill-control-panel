# Skill Control Panel 变更记录

## 1. 文档目的

这份文档用于按日期记录 `skill-control-panel` 从当前版本开始的所有功能变更。

记录原则：

- 每条变更都必须能对应到 `功能清单.md` 中的编号
- 每次迭代先更新功能清单，再写入本文件
- 同一功能可以被多次记录，但必须保留时间顺序

---

## 2. 记录格式

每次记录按日期归档，日期下使用以下分类：

- 新增
- 调整
- 删除
- 修复

每条记录必须包含：

- 对应功能编号
- 变更内容
- 必要时补充说明影响范围

---

## 3. 变更记录

## 2026-04-08

### 新增

- `FE-01` 建立首页概况区，统一展示去重 Skill 数、实例数、Provider 数和最近扫描时间。
- `FE-02` 建立 Skill 状态分布区，用于首屏展示当前系统中 Skill 的状态情况。
- `FE-03` 建立 Provider 分布展示，并收敛为只展示包含 Skill 的 Provider。
- `FE-04` 增加 Provider 快速筛选入口，使 Provider 分布区能直接带动列表筛选。
- `FE-05` 建立去重后的 Skill 卡片列表平铺展示结构。
- `FE-06` 明确 Skill 卡片的概览字段为名称、介绍、标签、状态、Provider。
- `FE-07` 增加关键词、状态、标签、Provider 四类筛选能力。
- `FE-08` 建立右侧固定详情栏作为桌面端默认详情交互。
- `FE-09` 建立 Skill 详情概览区，展示名称、介绍、标签、状态和真实路径。
- `FE-10` 建立 Provider 暴露详情区，逐项展示当前 Skill 的各 Provider 暴露路径。
- `FE-11` 建立 Provider 口径操作区结构，为后续启动、关闭、卸载能力预留按钮位置。
- `FE-12` 建立重新扫描入口。
- `FE-13` 建立首页空态与异常态文案结构。
- `BE-11` 明确 Provider 级 Skill 生命周期接口为后续待补能力，并在当前版本中保留接口缺口说明。

### 调整

- `BE-04` 当前前端展示侧仅消费有 Skill 的 Provider，不再把 0 Skill Provider 作为首页主要信息。

### 修复

- `FE-11` 当前版本不伪造 Provider 级操作能力，操作按钮默认不可用，并明确说明后端接口尚未接通。

## 2026-04-08（同日继续迭代）

### 新增

- `BE-06` 新增中文多标签推导能力，Skill 列表和详情开始统一消费 `tags[]`。
- `BE-11` 新增 Provider 级 Skill 启动、关闭、卸载真操作接口。
- `API-05` 新增 `POST /api/skills/:skillId/providers/:providerName/action` 接口，用于执行 Provider 级暴露操作。

### 调整

- `FE-01` 顶部大提示语占位被移除，首页改为紧凑工具栏。
- `FE-06` Skill 卡片改为展示全部 Provider，不再使用省略方式。
- `FE-07` 筛选区从左侧栏调整为列表顶部横向排布。
- `FE-08` 详情区宽度扩大，以适配更高频的查看与操作。
- `FE-11` Provider 口径操作区从占位结构升级为可执行真操作。
- `FE-12` `重新扫描` 按钮被收敛到顶部工具栏。

### 修复

- `FE-06` 卡片中的状态样式重新设计，视觉区分度明显增强。

## 2026-04-08（UI/UX 收敛）

### 调整

- `FE-01` 顶部概况区进一步压缩，降低“头重脚轻”问题，把更多纵向空间让给 Skill 列表。
- `FE-02` 状态分布从大块面板收敛为紧凑状态条，降低视觉噪音。
- `FE-03` Provider 分布移入列表工作区头部，兼顾展示和筛选，不再占用顶部主空间。
- `FE-06` Skill 卡片重组为标准 header / body / footer 结构，阅读路径更清晰。
- `FE-07` 横向筛选栏继续紧凑化，`清除全部` 改为弱化文本链接。
- `FE-08` 详情栏通过更强阴影和更深背景提升“聚焦层”感。
- `FE-11` 详情操作区按钮层级拉开，启动、关闭、卸载在视觉上区分主操作、次操作和危险操作。

### 修复

- `FE-06` 页面中“药丸泛滥”的问题被收敛，状态、标签、Provider 的视觉语义开始分离。

## 2026-04-08（视觉层级强化）

### 调整

- `FE-01` 页面底层背景改为更深一档的 slate 灰蓝系，以衬托卡片层。
- `FE-06` 卡片标题和描述文字进一步放大并加重，阅读效率提升。
- `FE-08` 详情区扩宽到 480px，并通过更强左侧阴影强化悬浮层次。

### 修复

- `FE-06` 卡片 hover 上浮和阴影增强后，界面“太平”的问题进一步缓解。

## 2026-04-08（Provider 主题联动）

### 调整

- `FE-03` 切换到不同 Provider 时，页面整体背景、主按钮、选中态和强调色开始联动为对应主题色。
- `FE-04` 当前 Provider 视角会在顶部扫描信息中明确显示，帮助用户第一时间感知当前筛选口径。

## 2026-04-08（Dashboard 收束）

### 调整

- `FE-01` 顶部独立数字卡被压缩并整合进列表头部的紧凑 stats rail，首屏把更多空间留给核心列表。
- `FE-03` Provider 标识从纯文字缩写升级为 SVG 图标系统，当前视角识别更快。
- `FE-06` Skill 卡片的信息层级继续收束，数字和路径开始采用更偏数据型的字体角色。

### 修复

- `FE-13` 异步操作的成功与失败提示补充了 `aria-live` / `role="alert"` 语义。
- `FE-06` 与 `FE-08` 的动效补充了 `prefers-reduced-motion` 降级，减少动画带来的可访问性问题。

## 2026-04-09

### 调整

- `FE-06` Skill 卡片与标签的表面层次进一步拉开，卡片改为更纯净的白色面层，标签和 Provider 使用不同灰度语义面。
- `FE-08` 详情区改为更独立的灰白聚焦面板，内部 Provider 卡片与外层容器的层级差更明显。
- `FE-07` 筛选条增加独立容器背景和边框，与列表区不再糊成同一层。

### 修复

- `FE-06` 卡片、标签和背景色过于相近导致的视觉黏连问题被明显缓解。

## 2026-04-09（品牌化与键盘反馈）

### 调整

- `FE-03` Provider 图标进一步升级为更接近品牌识别的矢量图，而非简单缩写或通用符号。
- `FE-01` stats rail 扩展为更信息密集的微型 dashboard，补充筛选结果数和标签覆盖数。
- `FE-08` 详情区和内部 Provider 卡片补充 `focus-within` 反馈，键盘导航时层级更明确。

### 修复

- `FE-08` 详情区键盘操作时缺少聚焦反馈的问题被修复。
- `FE-01` 顶部统计信息过于稀疏的问题被进一步收束。

## 2026-04-09（自测体系）

### 新增

- `QA-01` 新增后端回归测试入口，统一通过 `npm test` 验证核心后端能力。
- `QA-02` 新增基于 Chrome 原生 CDP 的前端冒烟测试，统一通过 `npm run test:ui` 验证前端交互链路。
- `API-05` 被纳入前端自测覆盖范围，验证 Provider 级暴露操作从 UI 到接口的实际联动。
