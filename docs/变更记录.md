# Skill Control Panel 变更记录

## 1. 文档目的

这份文档用于按日期记录 `skill-control-panel` 从当前版本开始的所有功能变更。

记录原则：

- 每条变更都必须能对应到 `功能清单.md` 中的编号
- 每次迭代先更新功能清单，再写入本文件
- 同一功能可以被多次记录，但必须保留时间顺序

---

## 2. 记录格式

每次记录按日期归档，日期下使用以下分类：

- 新增
- 调整
- 删除
- 修复

每条记录必须包含：

- 对应功能编号
- 变更内容
- 必要时补充说明影响范围

---

## 3. 变更记录

## 2026-07-12（私有 Skill Hub 适配）

### 新增

- `FE-19`：库存、详情和顶部统计新增 Hub 标签；Hub 与公共、Agent、项目可并列显示，筛选支持 Hub。
- `FE-20`：管理规范明确实体源、五个 Agent 发现入口及 Hub 更新后的路由应用命令；Hub 共享入口详情显示实现项目和入口同步约束。
- `FE-21`：环境诊断新增 Hub 当前事实摘要，并把待纳管项目/试用目录与需要处理的告警分组。

### 调整

- README、管理规范和聊天功能文案统一为“Hub 实体源 + 发现入口软路由”模型；底层 `public | agent | project` Registry 与 CLI 契约保持不变。
- Hub apply 现在将 Hub 实体统一标记为 `managed`；公共路由纳入受管白名单，支持安全移出 Hub 到项目层。

### 修复

- 修复公共 Hub Skill 的链式发现路由无法被 `move` 移除的问题；只有链条最终解析到原实体时才删除。

## 2026-07-11（Agent 官方与插件 Skill）

## 2026-07-11（安全与事务加固）

### 调整

- `BE-01`、`CLI-03`：项目层 Registry 和 CLI 都要求绝对 `project_root`，不再隐式使用控制台当前目录。
- `CLI-07`、`CLI-10`：public/Agent 路由按受管根和预期目标校验；移动时显式替换保留路由，锁释放只作用于本进程持有的 owner token。
- `UP-01`：远端更新地址仅接受受支持的 Git、HTTP(S) 和 SSH 形式，并在受限子路径中计算摘要。
- `FE-06`、`FE-07`：来源只允许 HTTP(S) 跳转；存在 `update.error` 时明确显示检查失败。

### 修复

- 修复 Registry 被篡改后，`update`、`move`、`uninstall` 可能作用于 scope 外路径的问题；写操作现在同时校验规范路径、realpath、稳定 ID 和 ownership。
- 修复 `SKILL.md` 符号链接、adopt 批处理与更新检查并发覆盖、缺失规范目录 reconcile、更新后过期状态、暂态错误 History 和本地来源误标“最新”等一致性问题。
- 修复错误目标路由被删除、整目录 public 路由遗漏、Claude 未知版本放行、move 沿用旧路径 ID、不可读扫描项拖垮全局等问题。
- 新增文件锁替换、并发过期锁回收、孤儿 guard、恶意来源 URL、路径逃逸、并发 check/update、事务 History 失败和前端协议过滤回归测试。
