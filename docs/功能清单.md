# Skill Control Panel 功能清单

## 1. 文档目的

这份文档用于维护 `skill-control-panel` 当前版本的功能台账。

它记录前端和后端所有已确认功能的当前状态，并作为后续迭代时的唯一功能对照基线。

维护原则：

- 每次迭代前先更新这份文档
- 所有变更必须能定位到具体功能编号
- 删除功能时保留编号，只调整状态，不直接移除条目

---

## 2. 当前版本基线

- 当前版本：`0.1.0`
- 建档日期：`2026-04-08`
- 当前文档角色：长期功能台账

## 3. 前端功能清单

### FE-01 首页概况区

- 状态：已上线
- 当前行为：首页通过列表头部的紧凑 stats rail 展示去重 Skill 数、Skill 实例数、活跃实例数、有 Skill 的 Provider 数，并通过紧凑工具栏展示最近扫描时间
- 关联位置：首页顶部概况区
- 备注：顶部概况已压缩，优先把屏幕高度让给 Skill 列表；stats rail 已进一步升级为更信息密集的微型 dashboard

### FE-02 Skill 状态分布

- 状态：已上线
- 当前行为：按 Skill 当前状态汇总展示状态分布，仅展示实际存在的状态
- 关联位置：Skill 列表头部紧凑状态条
- 备注：不展示告警信息，视觉上改为低噪音紧凑条

### FE-03 Provider 分布展示

- 状态：已上线
- 当前行为：只展示包含 Skill 的 Provider，并展示每个 Provider 的去重 Skill 数和实例数
- 关联位置：Skill 列表头部 Provider 快捷条
- 备注：0 Skill 的 Provider 不进入展示层；当切换到某个 Provider 视角时，页面整体主题色会同步切换，并配合更接近官方品牌的矢量图强化识别

### FE-04 Provider 快速筛选

- 状态：已上线
- 当前行为：点击 Provider 分布项可切换为对应 Provider 的列表筛选
- 关联位置：Provider 分布区、横向筛选栏
- 备注：筛选条件可清空；切换后会同步强化当前 Provider 的全局视觉上下文

### FE-05 Skill 列表平铺展示

- 状态：已上线
- 当前行为：以卡片方式平铺去重后的所有 Skill
- 关联位置：首页主工作区中栏
- 备注：桌面端为多列卡片，移动端降级为单列

### FE-06 Skill 卡片概览信息

- 状态：已上线
- 当前行为：每张卡片按 header / body / footer 结构展示 Skill 名称、介绍、中文标签、强化状态样式、所属 Provider
- 关联位置：Skill 卡片
- 备注：Provider 不再省略，全部直接展示，标签和 Provider 在视觉上已分层，并开始使用 SVG 图标系统；卡片、标签、Provider 与背景面层已进一步拉开

### FE-07 Skill 搜索与筛选

- 状态：已上线
- 当前行为：支持按关键词、状态、标签、Provider 进行筛选
- 关联位置：Skill 列表顶部横向筛选栏
- 备注：关键词搜索覆盖名称、简介、标签、Provider 名称

### FE-08 Skill 详情栏

- 状态：已上线
- 当前行为：点击卡片后在右侧固定详情栏展示当前 Skill 的完整信息
- 关联位置：首页右侧详情栏
- 备注：通过更强阴影、更独立的灰白底色和更清晰的内部卡片层强化为聚焦层，并补充 `focus-within` 的键盘聚焦反馈

### FE-09 Skill 详情概览

- 状态：已上线
- 当前行为：详情栏展示 Skill 名称、介绍、状态、标签、真实路径
- 关联位置：详情栏顶部
- 备注：作为进入操作前的基础信息确认区

### FE-10 Provider 暴露详情

- 状态：已上线
- 当前行为：详情栏逐项展示当前 Skill 在各 Provider 下的暴露信息和路径
- 关联位置：详情栏 Provider 暴露区
- 备注：包含 scope、是否为软链、暴露路径

### FE-11 Provider 口径操作区

- 状态：已上线
- 当前行为：详情栏可对每个 Provider 暴露执行“启动 / 关闭 / 卸载”真操作
- 关联位置：详情栏 Provider 暴露区
- 备注：按钮已做主次危险等级区分，卸载保留确认

### FE-12 重新扫描

- 状态：已上线
- 当前行为：用户可通过顶部工具栏按钮重新拉取当前环境数据
- 关联位置：首页顶部工具栏
- 备注：刷新后保留当前筛选条件

### FE-13 空态与异常态

- 状态：已上线
- 当前行为：覆盖加载中、加载失败、无 Provider、无 Skill、筛选无结果、未选中详情等状态
- 关联位置：首页各主要区块
- 备注：使用短句式提示文案，并为异步反馈补充可访问性播报

## 4. 后端功能清单

### BE-01 本地 HTTP 服务

- 状态：已上线
- 当前行为：提供本地服务并托管 dashboard 静态资源
- 关联位置：本地 server
- 备注：作为控制台唯一访问入口

### BE-02 Skill 扫描

- 状态：已上线
- 当前行为：通过现有扫描能力读取当前环境下的 Skill 实例信息
- 关联位置：扫描层
- 备注：输出原始 Skill 实例列表

### BE-03 去重 Skill 视图构建

- 状态：已上线
- 当前行为：按真实路径合并 Skill 实例，形成去重后的 Skill 列表
- 关联位置：数据整形层
- 备注：供前端列表与详情使用

### BE-04 Provider 汇总

- 状态：已上线
- 当前行为：按 Provider 汇总实例数和去重 Skill 数
- 关联位置：汇总层
- 备注：前端展示时仅消费其中有 Skill 的 Provider

### BE-05 Skill 元数据读取

- 状态：已上线
- 当前行为：读取 Skill 管理元数据，并在缺省时补默认值
- 关联位置：元数据层
- 备注：当前前端主要消费 `category` 和 `status`

### BE-06 中文标签推导

- 状态：已上线
- 当前行为：后端根据 Skill 名称、描述、目录名和分类规则推导中文 `tags[]`
- 关联位置：数据整形层
- 备注：标签数量不固定，以覆盖 Skill 功能为目标

### BE-07 重新扫描能力

- 状态：已上线
- 当前行为：支持重新执行扫描并返回最新汇总结果
- 关联位置：刷新接口
- 备注：供首页重新扫描按钮使用

### BE-08 Provider 启用/禁用

- 状态：已上线
- 当前行为：支持更新 Provider 的启用状态
- 关联位置：Provider 管理
- 备注：当前能力存在，但不作为本版前端首页主操作

### BE-09 Skill 安装

- 状态：已上线
- 当前行为：支持安装 Skill 并在应用后重新扫描
- 关联位置：安装能力
- 备注：当前能力存在，但不作为本版首页主流程

### BE-10 Skill 卸载

- 状态：已上线
- 当前行为：支持按 Skill 名和 scope 执行卸载
- 关联位置：卸载能力
- 备注：当前是已有能力，不等同于 Provider 级卸载

### BE-11 Provider 级 Skill 启动/关闭/卸载接口

- 状态：已上线
- 当前行为：支持对单个 Skill 在单个 Provider 下的暴露执行启动、关闭、卸载
- 关联位置：详情操作接口
- 备注：关闭通过本地持久化记录已停用暴露，支持后续恢复

## 5. 接口清单

### API-01 `GET /api/summary`

- 状态：已上线
- 当前行为：返回扫描时间、摘要信息和最近一次应用结果

### API-02 `GET /api/skills`

- 状态：已上线
- 当前行为：返回去重后的 Skill 列表

### API-03 `GET /api/providers`

- 状态：已上线
- 当前行为：返回 Provider 列表及其统计信息

### API-04 `POST /api/rescan`

- 状态：已上线
- 当前行为：触发重新扫描并返回最新摘要

### API-05 `POST /api/skills/:skillId/providers/:providerName/action`

- 状态：已上线
- 当前行为：执行 `enable | disable | uninstall` 三种 Provider 级 Skill 暴露操作

## 8. 维护规则

后续每次迭代必须遵守以下顺序：

1. 先修改本文件中的对应功能项
2. 再在 `变更记录.md` 中登记对应日期记录

功能项更新规则：

- 新增功能：新增编号，并在变更记录中记为“新增”
- 调整功能：修改对应条目的“当前行为”，并在变更记录中记为“调整”
- 删除功能：保留原编号，将状态改为“已移除”，并在变更记录中记为“删除”
- 修复行为：更新对应条目描述，并在变更记录中记为“修复”
